### A) Field-level constraints (type/required/unique/choices)

  - Type: `CharField(max_length=100)`

- **`username`**
  - **Required**
  - **Unique** (`unique=True`)
  - Type: `CharField(max_length=100)`
  - Validator: `_USERNAME_VALIDATOR`
    - **Regex**: `^(?=.*[A-Za-z])[A-Za-z0-9._]{3,32}$`
    - Meaning:
      - length 3–32
      - must contain **at least one letter**
      - allowed chars: letters, digits, dot `.`, underscore `_`

- **`email`**
  - **Required**
  - **Unique**
  - Type: `EmailField(unique=True)`
  - Extra strict validation in [clean()](cci:1://file:///home/fawad/Downloads/FYP-LETS-GO/Lets-Go/backend/lets_go/models/models_vehicle.py:81:4-89:90) (see below)

- **`password`**
  - **Required**
  - Type: `CharField(max_length=128)`
  - Validators:
    - `MinLengthValidator(8)` => minimum length **8**
  - Extra complexity rules in [clean()](cci:1://file:///home/fawad/Downloads/FYP-LETS-GO/Lets-Go/backend/lets_go/models/models_vehicle.py:81:4-89:90) (see below)

- **`address`**
  - **Required**
  - Type: `TextField()`

- **`phone_no`**
  - **Required**
  - Type: `CharField(max_length=16)`
  - Regex validator:
    - **Regex**: `^\+\d{10,15}$`
    - Must be international format with `+` and 10–15 digits
    - Example: `+923001234567` (Pakistan)

- **`cnic_no`**
  - **Required**
  - Type: `CharField(max_length=15)`
  - Regex validator:
    - **Regex**: `^\d{5}-\d{7}-\d{1}$`
    - Format: `36603-0269853-9`

- **`gender`**
  - **Required**
  - Choices: `male` or `female` (lowercase)
  - Type: `CharField(max_length=10, choices=[('male','Male'),('female','Female')])`

- **`driving_license_no`** (optional)
  - `null=True`, `blank=True`
  - Type: `CharField(max_length=20)`
  - Regex `_DL_NO_VALIDATOR`:
    - **Regex**: `^[A-Z0-9][A-Z0-9\-/]{4,19}$`
    - Meaning:
      - length 5–20
      - allowed: letters A-Z, digits, `-`, `/`
      - must start with `[A-Z0-9]`

- **Banking (optional but with cross-field rules)**
  - **`accountno`**
    - optional (`null=True`, `blank=True`)
    - Regex: `^\d{10,20}$` (10–20 digits)
  - **`bankname`**
    - optional (`null=True`, `blank=True`)
    - Regex: `^[A-Za-z][A-Za-z .&\-]{1,48}[A-Za-z.]$`
    - Meaning:
      - 2–50 chars
      - letters/spaces/`.`/`&`/`-`
      - must start with letter, end with letter or `.`
  - **`iban`**
    - optional (`null=True`, `blank=True`)
    - Regex: `^PK\d{2}[A-Z]{4}\d{16}$`
    - Pakistan IBAN format, **24 chars** like `PK36SCBL0000001123456702`

- **Image URL fields** (optional)
  - `profile_photo_url`, `live_photo_url`, `cnic_front_image_url`, `cnic_back_image_url`,
  - `driving_license_front_url`, `driving_license_back_url`, `accountqr_url`
  - All are `URLField(null=True, blank=True)` (so they must be valid URLs if provided)

- **`status`**
  - Choices: `PENDING`, `VERIFIED`, `REJECTED`, `BANNED`
  - Default: `PENDING`

### B) [clean()](cci:1://file:///home/fawad/Downloads/FYP-LETS-GO/Lets-Go/backend/lets_go/models/models_vehicle.py:81:4-89:90) cross-field rules + normalization (critical)
These rules run on `full_clean()`.

- **Normalization**
  - `username = username.strip()`
  - `email` is stripped (but not lowercased here; in views you lower it)
  - `driving_license_no`:
    - strip
    - uppercase
    - remove spaces
  - `accountno`: strip and remove spaces
  - `iban`: strip, uppercase, remove spaces
  - `bankname`: strip

- **Email extra strict validation**
  - Regex used (domain + TLD strict):
    - `^[^@\s]+@([A-Za-z0-9](?:[A-Za-z0-9-]{0,61}[A-Za-z0-9])?)(?:\.([A-Za-z0-9](?:[A-Za-z0-9-]{0,61}[A-Za-z0-9])?))*\.[A-Za-z]{2,24}$`
  - Rejects emails like `a@b` or invalid domains.

- **Password complexity**
  Password must contain:
  - at least one uppercase: `[A-Z]`
  - at least one lowercase: `[a-z]`
  - at least one digit: `\d`
  - at least one special char from:
    - `[!@#$%^&*()_+\-=\[\]{};':\"\\|,.<>\/?]`

- **Driving license images requirement (Pakistan-specific KYC)**
  - If `driving_license_no` is provided:
    - **both** `driving_license_front_url` and `driving_license_back_url` must be present
  - Note: this is a *hard requirement* currently.

- **Bank name required when bank details present**
  - If either `accountno` **or** `iban` is provided:
    - `bankname` is required
